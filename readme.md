### da213: synthetic data & sampling effects
this repo contains the codebase for our DA213 project (Project 3). we are testing how different sampling strategies impact the quality of synthetic data generated by a gaussian copula model.

authors: rup narayan jha || anuj kumar || jaypal malviya

### what this does
we took the brfss 2015 diabetes dataset and ran it through 7 different sampling methods. for each method, we grab a subset of data, train an sdv gaussian copula, generate fake rows, and then score how "real" the synthetic data actually looks.

we loop this 10 times per method to get stable averages and avoid random flukes.

### methods we tested:

pure random
unified sequence (linear + quadratic jumps)
cluster (biased targeting on education levels)
stratified (proportional splits by age)
boundary (forcing the model to learn extreme bmi edges)
conditional logic (restricting to high bp & high chol)
weighted (boosting rare heart disease cases)

### how we scored it
instead of relying on one metric, we hit the synthetic data with three different tests:

sdv metrics: standard library checks for column shapes and pair trends.
wasserstein distance: checking the 1d distribution shift on continuous variables like bmi.
the turing test (discriminator auc): we built a logistic regression model, fed it half real and half fake data, and saw if it could tell the difference. (an auc closer to 0.50 means the fake data is highly realistic).
repo structure
data/raw/ -> drop the brfss 2015 csv here. (it's ignored in git because the file is too big).
notebooks/ -> 01_sampling_evaluation.ipynb is the main orchestration file that runs the 70-loop pipeline.
src/ -> the core logic. sampling_methods.py has the algorithms, synthetic_gen.py wraps the model, and quality_metrics.py handles the scoring and dashboard generation.
outputs/ -> execution logs, final synthetic csv files, and visual dashboards get dumped here.

### how to run
clone the repo.
put diabetes_binary_health_indicators_BRFSS2015.csv into data/raw/.
install dependencies: pip install -r requirements.txt
open the notebook and run it top to bottom. (warning: running the full 10 iterations takes about 10-15 minutes depending on your cpu).
